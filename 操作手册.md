## 一、Git 工作流：同步上游 & 推送你的 fork

你的 fork 是 `origin`，官方仓库是 `upstream`。`git push origin main` 只会推到你的 fork，不会影响 upstream。

### 1.1 日常同步上游

当 upstream（`0xSero/codex-shim`）有新提交时：

```bash
cd /Users/maerun/Documents/Third-Party/codex-shim

# 1. 拉取上游最新提交
git fetch upstream

# 2. 查看上游领先了哪些 commit
git log --oneline HEAD..upstream/main

# 3. 把上游合并到你的 main
git checkout main
git merge upstream/main

# 4. 推到你的 fork
git push origin main
```

如果合并后有冲突，按编辑器提示解决 → `git add <冲突文件>` → `git commit`。

### 1.2 把开发分支变基到最新 main

如果你在某个功能分支上工作（比如之前的 `feature/smart-vision-routing`），同步完 main 后需要变基：

```bash
git checkout <你的功能分支>
git rebase main
```

rebase 过程中如果遇到冲突：
- 解决冲突 → `git add <冲突文件>` → `git rebase --continue`
- 想放弃：`git rebase --abort`

rebase 改写了 commit hash，推送时需要强制推送：

```bash
git push origin <你的功能分支> --force-with-lease
```

> `--force-with-lease` 比 `--force` 更安全：如果远端有你不知道的新提交，它会拒绝推送，避免覆盖别人的工作。

### 1.3 合并后根据改动决定后续动作

```bash
# 看上游具体改了哪些文件
git diff HEAD~1 HEAD -- codex_shim/
```

| 上游改了什么 | 后续动作 |
|---|---|
| `cli.py` 里 needle / patch 逻辑 | 需要 `restore-app` + `patch-app` |
| `server.py` / `catalog.py` / `settings.py` / 路由逻辑 | 需要 `codex-shim restart` |
| `tests/` / `README.md` / `docs/` | 无需任何动作 |
| `pyproject.toml` 的 runtime 依赖 | 重跑 `pip install --user -e ".[dev]" --break-system-packages` |

### 1.4 合并后跑测试

```bash
python3 -m pytest -x -q
```

### 1.5 紧急回滚

如果合并上游后引入问题：

```bash
git reflog | head -20          # 找到合并前的 commit
git reset --hard HEAD@{N}      # 回到那个状态
git push origin main --force-with-lease
```

## 二、codex-shim 日常操作

### 2.1 models.json 配置

配置文件位于 `~/.codex-shim/models.json`。当前配置 8 个模型 + Auto Router：

**模型列表**：

| slug | 通道 | 用途 |
|---|---|---|
| `deepseek-v4-flash` | DeepSeek 官方 | 便宜快速文本 + 分类器 |
| `deepseek-v4-pro` | DeepSeek 官方 | 复杂文本兜底 |
| `deepseek-v4-flash-ark` | Ark coding plan | 便宜快速文本（优先） |
| `deepseek-v4-pro-ark` | Ark coding plan | 复杂文本（优先） |
| `gpt-5-5` | freemodel.dev | 多模态通用 |
| `glm-5.1` | Ark coding plan | 多模态通用 |
| `kimi-k2-6` | Ark coding plan | 含图任务（第三选择） |
| `doubao-seed-2-0-pro` | Ark coding plan | 含图任务（第一选择） |
| `minimax-m2-7` | Ark coding plan | 多模态通用 |

**Auto Router**（slug `codex-auto`，display name `Auto (smart routing)`）：
- 分类器用 `deepseek-v4-flash`（官方通道，不依赖 Ark 额度）
- 阈值 0.7，过线的候选里挑最便宜的
- 兜底走 `deepseek-v4-flash-ark`（Ark 通道）
- 含图任务自动剔除 `supports_images: false` 的候选，按 cost 排序：doubao (1.2) → gpt-5.5 (1.5) → kimi-k2.6 (2.0)

**添加新模型**的字段说明：

```json
{
  "model": "模型在 provider 侧的 ID",
  "provider": "openai | anthropic | generic-chat-completion-api",
  "base_url": "provider 的 API 端点",
  "api_key": "你的 API key",
  "display_name": "在 Codex picker 里显示的名称",
  "max_context_limit": 128000,
  "no_image_support": true,
  "max_output_tokens": 4096
}
```

**编辑完 models.json 后**：

```bash
codex-shim generate        # 重新生成 catalog（codex-auto 也会写入）
codex-shim restart         # 重启守护进程加载新配置
```

### 2.2 启动 & 停止

```bash
cd /Users/maerun/Documents/Third-Party/codex-shim

codex-shim generate        # 读取 models.json，生成 catalog + config
codex-shim start           # 在 127.0.0.1:8765 启动守护进程
codex-shim status          # 健康检查 + 模型数量
codex-shim list            # 列出所有模型 slug 与上游路由
codex-shim stop            # 停止守护进程
codex-shim restart         # 停止 → 重新生成 → 重启
```

生成的文件在 `.codex-shim/` 下（已 gitignore）：

```text
.codex-shim/custom_model_catalog.json   # Codex 模型选择器目录
.codex-shim/config.toml                 # Codex provider 配置
.codex-shim/shim.pid                    # 守护进程 PID
.codex-shim/shim.log                    # 日志
```

### 2.3 接入 Codex Desktop

```bash
codex-shim app .           # 启动 Codex Desktop 并接入 shim
codex-shim disable         # 撤销，恢复原配置
```

`app` 会自动生成目录、启动守护进程、向 `~/.codex/config.toml` 写入受管理块。

### 2.4 切换模型

```bash
codex-shim model list              # 查看可选模型
codex-shim model use <slug>        # 设为 Desktop 默认模型
codex-shim app .                   # 以新默认模型重启 Desktop
```

### 2.5 CLI 临时调用（不改配置）

```bash

```

### 2.6 Desktop 选择器补丁（patch-app）

Codex Desktop 默认隐藏自定义模型。打补丁后才能在选择器里看到。

```bash
# 打补丁
sudo "$(which codex-shim)" patch-app

# 回滚补丁
sudo "$(which codex-shim)" restore-app
```

打完补丁后重启 Desktop：

```bash
osascript -e 'tell application "Codex" to quit' 2>/dev/null
killall Codex 2>/dev/null
sleep 1
open /Applications/Codex.app
```

**如果 patch-app 失败**（提示 "Could not find the expected model picker allowlist filter"），说明新版 Desktop 改了前端代码，需要侦察：

```bash
# 解包 app.asar
cd /tmp && rm -rf codex-asar-inspect && mkdir codex-asar-inspect && cd codex-asar-inspect
npx --yes @electron/asar extract /Applications/Codex.app/Contents/Resources/app.asar extracted

# 定位过滤逻辑
grep -RIln 'amazonBedrock' extracted/webview/assets/

# 把候选文件美化导出到仓库内分析
TARGET_DIR=/Users/maerun/Documents/Third-Party/codex-shim/.tmp-bundle-inspect
rm -rf "$TARGET_DIR" && mkdir -p "$TARGET_DIR"
for f in $(grep -RIln 'amazonBedrock' extracted/webview/assets/*.js 2>/dev/null); do
  name=$(basename "$f")
  npx --yes prettier "$f" > "$TARGET_DIR/${name}.pretty.js"
done
```

然后修改 `codex_shim/cli.py` 里的 `MODEL_PICKER_NEEDLE` / `MODEL_PICKER_REPLACEMENT` / `MODEL_PICKER_APPLIED` 三个常量，再重试。

### 2.7 Auto Router 开关

```bash
# 开启（带路由决策日志）
CODEX_SHIM_ROUTER_LOG=1 codex-shim restart

# 急停（不改配置）
CODEX_SHIM_DISABLE_ROUTER=1 codex-shim restart

# 恢复
CODEX_SHIM_ROUTER_LOG=1 codex-shim restart
```

禁用后 `shim-auto` 从 picker 消失，所有请求按你手动选的模型走。

> **注意**：Router 的 slug 必须避开 `codex-*` 前缀（默认是 `shim-auto`）。Codex Desktop 0.133+ 对 `codex-auto-review` 等 `codex-auto*` slug 有内部路由表，会绕过 codex_shim provider，导致请求发不到本地 shim 直接报 `error sending request for url`。

### 2.8 查看日志

```bash
tail -f /Users/maerun/Documents/Third-Party/codex-shim/.codex-shim/shim.log
```

开启 Router 日志后，每次对话会打印决策信息：

```text
[router] -> deepseek-v4-flash-ark (score=0.91; score>=0.70, cheapest) scores={...}
[router] codex-auto -> deepseek-v4-flash-ark
[route] slug='deepseek-v4-flash-ark' provider='openai' upstream_model='deepseek-v4-flash' url=...
```

## 三、Codex Desktop 升级后的修复流程

Codex Desktop 升级会覆盖 `app.asar`，清掉之前的补丁。升级后执行：

```bash
cd /Users/maerun/Documents/Third-Party/codex-shim

# 1. 顺手看上游有没有新东西
git fetch upstream
git log --oneline HEAD..upstream/main

# 2. 恢复 config（如果被升级器洗掉了）
grep -A 5 codex_shim ~/.codex/config.toml || codex-shim enable

# 3. 重新打补丁
sudo "$(which codex-shim)" restore-app
sudo "$(which codex-shim)" patch-app

# 4. 重启 Desktop
osascript -e 'tell application "Codex" to quit' 2>/dev/null
killall Codex 2>/dev/null
sleep 1
open /Applications/Codex.app
```

如果 patch-app 失败，回到 2.6 节的侦察流程。

## 四、常见问题速查

| 现象 | 处理 |
|---|---|
| 端口被占用 | `codex-shim --port 8766 restart` |
| `codex-shim list` 显示 "No models" | 检查 `~/.codex-shim/models.json` 是否存在；或 `codex login` |
| picker 只显示 `default` | 先 `codex-shim generate`，再看 patch-app 是否有效 |
| 模型出现但请求 404 | `codex-shim generate` → `codex-shim model use <slug>` |
| 流式响应卡住 | `codex-shim restart`，看 `shim.log` |
| 补丁后 Codex 崩溃 | `codex-shim restore-app` → `codex-shim patch-app` |
| `git push` 报 non-fast-forward | rebase 后必须用 `--force-with-lease` |
| Ark 配额耗尽，Auto 仍选 Ark | 手动切官方模型，或 `CODEX_SHIM_DISABLE_ROUTER=1 codex-shim restart` |
| Desktop 选 Auto 报 `error sending request for url` | Router slug 用了 `codex-*` 前缀，与 Desktop 内置 `codex-auto-review` 冲突。改 `~/.codex-shim/models.json` 的 `router.slug` 为 `shim-auto`（或任意非 `codex-*` 前缀），再 `codex-shim restart && codex-shim app .` |
| shim.log 为空但进程在跑 | Python stdout 缓冲。已修复（start 时注入 `PYTHONUNBUFFERED=1`），如果还遇到，前台直接跑 `python3 -m codex_shim.server ...` 验证 |
| `codex-shim status` 显示 health check failed 但 curl `/health` 正常 | Python 的 `urllib.request.urlopen` 在 3.14 下连 127.0.0.1 偶发失败。已修复为 `http.client.HTTPConnection` 直连 |
| 守护进程僵死 | `codex-shim restart`；仍不行删 `.codex-shim/shim.pid` 后 `codex-shim start` |
| `Operation not permitted` 写不进 `/Applications` | 系统设置 → 隐私与安全性 → App 管理，给终端开权限 |

### 应急路径（不依赖 Desktop 补丁）

如果 patch-app 一时修不好，可以完全绕开 Desktop UI：

```bash
# 浏览器选择器
open http://127.0.0.1:8765/picker

# CLI 直接跑
codex-shim codex -- "你的提示词"

# 固定默认模型
```
