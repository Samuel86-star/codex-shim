# codex-shim 完整操作命令手册（macOS）

本手册整理了 codex-shim 项目在 **macOS** 下的所有操作命令，按使用场景分类，方便快速查阅。

> 适用版本：Codex Desktop `0.133.0-alpha.1`（macOS arm64 测试通过）。shim 服务端为纯 Python/aiohttp。

---

## 零、本地环境约定（你的 fork 工作流）

本仓库是从上游 `0xSero/codex-shim` fork 出来的二次开发版本，本地具体设置：

| 项 | 值 |
|---|---|
| 仓库路径 | `/Users/maerun/Documents/Third-Party/codex-shim` |
| `origin` | `https://github.com/Samuel86-star/codex-shim.git`（你的 fork） |
| `upstream` | `https://github.com/0xSero/codex-shim.git`（官方原仓库） |
| 当前开发分支 | `feature/smart-vision-routing` |
| 安装模式 | `pip install -e .`（editable，源码即时生效，无需"编译"） |

基于这套设置，以下两条规则贯穿整本手册：

1. **改完代码不需要重新安装**。editable 模式让 `~/Library/Python/3.x/bin/codex-shim` 指回仓库源码，改 `codex_shim/*.py` 立即生效；仅当修改了 `pyproject.toml` 的依赖或入口点定义时，才需要重新跑一次 `pip install -e ".[dev]" --break-system-packages`。
2. **固定在仓库根目录运行 shim 生命周期命令**。`codex-shim generate / start / stop / restart / status` 会把 `.codex-shim/`（catalog、config、pid、log）写在当前工作目录下，不固定在仓库根目录跑会出现"start 和 stop 找不到同一个 pid 文件"的尴尬。约定如下：

```bash
cd /Users/maerun/Documents/Third-Party/codex-shim    # 所有 shim 管理命令前先 cd 过来
```

业务侧调用（`codex-shim app <项目路径>`、Codex Desktop、CLI）不受此约束 —— 它们走 `127.0.0.1:8765` 本地端口，从任意目录调用同一个守护进程。

### 与上游保持同步的常用 git 命令

```bash
git fetch upstream                       # 拉取官方最新提交
git checkout main
git merge upstream/main                  # 把官方主干合并到本地 main
git push origin main                     # 推到你的 fork

git checkout feature/smart-vision-routing
git rebase main                          # 把开发分支变基到最新 main
git push origin feature/smart-vision-routing --force-with-lease
```

### 切换分支后的快速自检

切换或新建分支后无需重装，跑一次 status 验证当前代码可启动即可：

```bash
git checkout <branch>
codex-shim status                        # 守护进程仍在 → 直接复用
codex-shim restart                       # 想用新分支代码热重启时执行
```

## 二、安装命令

> **一次性动作**。本仓库已通过 editable 模式装好，日常改代码、切分支都**无需重装**。仅当 `pyproject.toml` 中的依赖或入口点变化时，才回到本节重新执行。

```bash
git clone https://github.com/Samuel86-star/codex-shim.git ~/codex-shim   # 首次克隆你的 fork
cd ~/codex-shim
git remote add upstream https://github.com/0xSero/codex-shim.git         # 关联官方仓库
python3 -m pip install --user -e ".[dev]" --break-system-packages
```

参数说明：

- `--user`：安装到用户目录（`~/Library/Python/3.x/` 或 `~/.local/`），不污染系统 site-packages
- `-e .`：以可编辑模式（editable）安装当前仓库，源码改动立即生效，无需重新安装
- `".[dev]"`：在运行时依赖（`aiohttp`）之外，同时安装开发/测试依赖（`pytest`、`pytest-asyncio`）
- `--break-system-packages`：绕过 PEP 668 的 "externally-managed-environment" 限制。Homebrew Python 和较新版 macOS 系统 Python 都需要此参数，否则会拒绝安装

安装后会自动生成 `codex-shim` 命令入口。先确认安装成功并定位入口点目录：

```bash
python3 -m pip show codex-shim          # 验证已安装
python3 -m site --user-base             # 查看 --user 安装的根目录
echo "$(python3 -m site --user-base)/bin"   # 拼出 bin 目录（即 codex-shim 所在位置）
```

> macOS 下 `--user` 实际安装到 `~/Library/Python/3.x/`，而不是 Linux 常见的 `~/.local/`，因此用 `python3 -m site --user-base` 动态取路径比硬编码更可靠。

将该 `bin` 目录写入 PATH（以 zsh 为例）并立即生效：

```bash
echo 'export PATH="$(python3 -m site --user-base)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

验证命令可用：

```bash
which codex-shim
codex-shim --help
```

## 三、快速上手

### 1. 生成目录并启动 shim

```bash
codex-shim generate          # 读取 ~/.codex-shim/models.json（如存在）
codex-shim start             # 在 127.0.0.1:8765 启动后台守护进程
codex-shim list              # 显示生成的模型 slug 与上游路由
codex-shim status            # 健康检查 + 模型数量
```

生成的运行时文件位于仓库根目录下的 `.codex-shim/`：

```text
.codex-shim/custom_model_catalog.json   # Codex 模型选择器目录
.codex-shim/config.toml                 # 可选的 Codex provider 配置
.codex-shim/shim.pid                    # 守护进程 PID
.codex-shim/shim.log                    # 标准输出/错误 + 请求摘要
```

### 2. 将 Codex Desktop 指向 shim

```bash
codex-shim app .             # 启动 Codex Desktop 并接入 shim
```

`app` 命令会生成目录、按需启动守护进程，并向 `~/.codex/config.toml` 写入受管理块。原配置会备份到 `.codex-shim/` 下。撤销命令：

```bash
codex-shim disable
```

### 3. 切换 Desktop 当前激活模型

```bash
codex-shim model list
codex-shim model use gpt-5.5      # 或 list 中显示的任意 slug
codex-shim app .                  # 以新默认模型重新启动 Codex
```

### 4. 用 Codex CLI 临时调用（不修改配置）

```bash
codex-shim codex -- "分析此仓库并总结架构"
```

## 四、完整命令清单

### 主命令 `codex-shim`

```text
codex-shim generate          重新生成目录/配置（不启动守护进程）
codex-shim start             重新生成目录并启动本地 shim 守护进程
codex-shim enable            启动守护进程并向 ~/.codex/config.toml 写入受管理块
codex-shim status            健康检查 + 模型数量
codex-shim stop              停止守护进程
codex-shim disable           移除受管理配置块并停止守护进程
codex-shim restart           停止、重新生成、重启守护进程
codex-shim list              列出生成的 slug 与上游路由
codex-shim model list        列出选择器当前可用的 slug
codex-shim model use <slug>  在受管理配置中设置 Desktop 默认模型
codex-shim codex -- <args>   通过内联 shim 覆盖执行 codex CLI
codex-shim app [path]        通过受管理 shim 配置启动 Codex Desktop
codex-shim patch-app         为 Codex Desktop 选择器打补丁
codex-shim restore-app       从备份还原 app.asar
```

### 快捷命令

```text
codex-app [path]             等价于 codex-shim app
codex-model [list|<slug>]    等价于 codex-shim model …
```

### 全局参数

- `--settings <path>`：用于 catalog/model/start/app/codex 等流程，指定自定义模型配置文件
- `--port <port>`：用于守护进程 / provider 相关流程，更改监听端口

> `patch-app` 与 `restore-app` 始终以 `/Applications/Codex.app` 为目标，不接受 `--settings`。

## 五、自定义配置文件

默认读取 `~/.codex-shim/models.json`，也可通过 `--settings` 指定其他路径：

```bash
codex-shim --settings /path/to/my-models.json generate
codex-shim --settings /path/to/my-models.json start
```

推荐 schema 示例：

```json
{
  "models": [
    {
      "model": "gpt-5.5",
      "provider": "openai",
      "base_url": "https://api.openai.com/v1",
      "api_key": "sk-…",
      "display_name": "OpenAI GPT-5.5",
      "max_context_limit": 400000
    },
    {
      "model": "claude-opus-4-7-20251109",
      "provider": "anthropic",
      "base_url": "https://api.anthropic.com/v1",
      "api_key": "sk-ant-…",
      "display_name": "Claude Opus 4.7"
    },
    {
      "model": "deepseek-v4-pro",
      "provider": "anthropic",
      "base_url": "https://api.deepseek.com/anthropic",
      "api_key": "…",
      "display_name": "DeepSeek V4 Pro",
      "no_image_support": true
    }
  ]
}
```

支持的 `provider` 取值：

| provider | 上游 API |
|---|---|
| `openai` | OpenAI `/v1/chat/completions` |
| `generic-chat-completion-api` | OpenAI 形态的 chat completions |
| `anthropic` | Anthropic `/v1/messages` |

常用模型字段：

| 字段 | 行为 |
|---|---|
| `display_name` | 选择器中的人类可读名称 |
| `max_context_limit` | 目录上下文窗口与压缩阈值 |
| `max_output_tokens` | 翻译为 Anthropic 时的默认输出上限 |
| `no_image_support` | 为 true 时，目录声明仅支持纯文本输入 |
| `extra_headers` | 合并到请求中的额外上游 header |

加载器也接受 camelCase 别名（`baseUrl`、`apiKey`、`displayName`、`maxContextLimit`、`maxOutputTokens`、`noImageSupport`、`extraHeaders`）以及旧的顶层 `customModels` 数组。

### Ollama / 本地 OpenAI 兼容端点示例

```json
{
  "models": [
    {
      "model": "llama3.2",
      "display_name": "Ollama Llama 3.2",
      "provider": "generic-chat-completion-api",
      "base_url": "http://127.0.0.1:11434/v1",
      "api_key": "ollama"
    }
  ]
}
```

`provider: "ollama"` 会被自动规范化为 `generic-chat-completion-api`，在未指定 base URL 时默认为 `http://127.0.0.1:11434/v1`。

## 六、Desktop 选择器补丁

Codex Desktop 有服务端下发的允许列表（`use_hidden_models: true`），自定义目录条目会被隐藏。可通过 ASAR 补丁关闭这一过滤。

> 操作前务必备份 `app.asar` 与 `Info.plist`。

### 一键命令（推荐）

```bash
codex-shim patch-app
codex-shim restore-app
```

### 手动完整补丁流程

```bash
APP=/Applications/Codex.app
sudo cp -R "$APP" "$APP.unpatched-$(date +%Y%m%d-%H%M%S)"

# 1. 解包 ASAR
cd /tmp && rm -rf codex-asar-patch && mkdir codex-asar-patch && cd codex-asar-patch
npx --yes @electron/asar extract "$APP/Contents/Resources/app.asar" extracted

# 2. 改写选择器过滤器
PATCH_FILE=$(grep -RIl 'useHiddenModels' extracted/webview/assets/model-queries-*.js | head -n1)
sed -i.bak -E 's/let u=c\.useHiddenModels&&o!==`amazonBedrock`,d;/let u=!1,d;/' "$PATCH_FILE"
rm "$PATCH_FILE.bak"

# 3. 改写侧栏会话 provider 过滤器（详见 README 中的 Python 脚本）

# 4. 重打包
npx --yes @electron/asar pack extracted app.asar.new
sudo cp app.asar.new "$APP/Contents/Resources/app.asar"

# 5. 重新计算 header hash
HEADER_HASH=$(python3 - "$APP/Contents/Resources/app.asar" <<'PY'
import struct, hashlib, sys
with open(sys.argv[1], 'rb') as f:
    data_size, header_size, _, json_size = struct.unpack('<4I', f.read(16))
    header_json = f.read(json_size)
print(hashlib.sha256(header_json).hexdigest())
PY
)

# 6. 更新 Info.plist
sudo /usr/libexec/PlistBuddy -c \
  "Set :ElectronAsarIntegrity:Resources/app.asar:hash $HEADER_HASH" \
  "$APP/Contents/Info.plist"

# 7. ad-hoc 重新签名
sudo codesign --force --deep --sign - "$APP"

# 8. 启动
open "$APP"
```

回滚：

```bash
sudo rm -rf "$APP" && sudo mv "$APP.unpatched-…" "$APP"
```

## 七、Web 模型选择器

shim 内置一个浏览器小工具，便于无需重启 CLI 切换模型：

- `GET /picker` — 自包含的暗色主题 HTML 页面
- `GET /api/models` — 选择器对应的 JSON 列表
- `POST /api/switch` — 请求体：`{"slug": "...", "restart_codex": true|false}`

所有 picker 路由都受同样的 `Host` header 白名单保护，可防止 DNS rebinding 攻击。

## 八、常见排错命令

### shim 启动失败

```bash
codex-shim status
tail -n 80 .codex-shim/shim.log
```

端口被占用时换端口：

```bash
codex-shim --port 8766 restart
codex-shim --port 8766 app .
```

### `~/.codex-shim/models.json` 缺失

```bash
codex-shim --settings /path/to/my-models.json generate
```

### `codex-shim list` 显示 "No models available"

```bash
codex login                       # 生成 ~/.codex/auth.json
# 或
codex-shim --settings /path/to/my-models.json list
```

### Codex 只显示 `default`

```bash
codex-shim generate
codex-shim model list
```

如目录已包含模型但 Desktop 仍隐藏，请应用补丁。

### 模型出现但请求返回 404

```bash
codex-shim generate
codex-shim model list
codex-shim model use <slug>
```

### 流式响应卡住

```bash
codex-shim restart
tail -f .codex-shim/shim.log
```

### 补丁后 Codex 崩溃

```bash
codex-shim restore-app
codex-shim patch-app
```

### 重置 shim 全部生成状态

```bash
codex-shim stop
# 如需完全重置，手动删除 .codex-shim 目录
codex-shim generate
codex-shim start
```

## 十一、附录：相关说明

- **安全性**：shim 默认绑定 `127.0.0.1`，并对每个请求校验 `Host` header；API key 不会被复制进生成的目录文件
- **隐私**：默认日志只保留摘要级别，不输出完整 prompt / API key
- **ChatGPT 直通**：仅当 `~/.codex/auth.json` 包含有效 `tokens.access_token` 时启用，并将 token 仅转发到 ChatGPT 的 Codex 端点
- **`CODEX_SHIM_ALLOWED_HOSTS`** 环境变量：当 shim 故意绑定到非回环 host 时，将允许访问的 host 以逗号分隔列在此环境变量中

更多设计细节、路由原理与基准测试方法请参考仓库根目录下的 `README.md`、`VISION_ROUTING_CN.md` 与 `IMPLEMENTATION_SUMMARY_CN.md`。
